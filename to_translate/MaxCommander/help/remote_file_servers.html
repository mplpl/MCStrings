Using remote file servers

With the MaxCommander, you can access and manage files on remote file servers.

FTP/SFTP

FTP (and its secure variants FTPS and FTPES) and SFTP are supported natively by the MaxCommander app. After making a connection you will be able to access files from the MaxCommander app but not from other apps (including the Finder). When you exit the MaxCommander app, the connection will be terminated.

MaxCommander only supports FTP passive mode. FTP/SFTP server character encoding must be UTF-8.

SMB/AFP/WebDAV

SMB, AFP and WebDAV are supported over the operating system. After making a connection, a volume is mounted on the operating system level and stays mounted until you unmount it in the MaxCommander app or the Finder. You can access files on such volumes from the Finder even after the MaxCommander app quits.

URL format

When mounting a remote volume you need to provide its access URL, which is in the following format:

protocol://username@hostname:port/path

Attribute

Description
protocolThe mandatory attribute that must be selected from the drop-down list.
usernameThe optional attribute, when omitted, the name of the user who is running the app will be used. If this attribute is omitted, following “@” must also be omitted.
hostnameThe mandatory attribute – it can be a domain name or IP address. If IPv6 is used, it should be given in square brackets - “[“ and “]”.
portThe optional attribute – if omitted, the default port for the given protocol will be used. In such case, preceding “:” should also be omitted.
pathThe optional attribute. It determines remote path that will be set after connecting to the server. If omitted, also preceding “/” should be omitted. This path may be absolute (when it starts from “/”) or relative. In the latter case, it relates to a default folder of the remote server (typically it is the given user home folder).

For example, in “ftp://admin@127.0.0.1/test_dir”, the path is relative and is “test_dir”, which most likely means that subfolder “test_dir” of “admin” user home folder will be selected. But if the URL is “ftp://admin@127.0.0.1//test_dir” then “/test_dir” will be used.

“path” is only supported for FTP/SFTP.
A password must not be given in the URL. It will be retrieved from the system Keystore and if not present there, you will be asked to provide it.
