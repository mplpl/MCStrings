Selecting items

Normal and hard selection

Effective items selection is crucial for using the MaxCommander app as selected items are subjects of the file management operation like copying or deleting. There are two types of selection:
  • (Normal) selection – allows to only select one item
  • Hard selection – allows to select multiple items from the same folder
For simplicity, in this document we will use a term 'selection' instead of 'normal selection'.

Selecting items

To select an item, you just click the item or move the cursor to that item using arrows. Such selection happens all the time when you are browsing folders in the MaxCommander app and there is always one item that is selected.

Changing selection does not impact the set of hard selected items.

Selecting with quick find

You can select an item by quickly typing its name (or the begining of the name). When you do that, the first item which name start form the prefix you typed will be selected. Letter case is not taken into account.

Hard selecting items

To hard-select an item, select it and then press Shift + Down arrow (or Control + Space). This will hard-select the item and move the selection down. To hard-select and move the selection up, press Shift + Up arrow. If an item is already hard-selected, using one of these methods will unselect it.

To select an item that is a folder and count its size, press Option + Space.

You can also hard-select multiple items at once: select an item, then press Shift and use your mouse to click another item, and all the items between them will be hard selected.

To hard-select all the items, press Command + a.

To unselect all the items, press Control + n.

You can invert the current hard-selection press * – items that are hard-selected will become unselected, and these unselected will be hard-selected.

You can also use wildcards to extend or shrink the hard-selection – press + or - key.

Selection and operations

In general, operations like copying, moving, deleting, etc. take into account the list of hard selected items from the current panel and if none is hard select, the item that is normally selected is taken.

There are a few exceptions made for operation that only takes single files as inputs. Such operations ignore hard selection and only use the normally selected item instead. This includes:
  • Running/opening an item (Enter)
  • Displaying an item information (Command + i)
  • Rename (Control + r)
  • Comparing items by content – it only takes into account selected items in both panels
  • Comparing MD5 - it only takes into account selected items in both panels

Selection and an item context menu

When requesting an operation from item’s context menu (it opens when you click an item with the right mouse button) there is another aspect that needs to be taken into account – right-clicked item may be neither normal nor hard selected. Therefore, for operations requested from that menu:
  • If the clicked item is not hard selected or when an operation only supports single items, the clicked item will be the subject for the operation
  • Otherwise, all the hard selected items are the subject for the operation
