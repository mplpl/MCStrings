Searching for items

Quick find

To find and select an item in the current folder, quickly start typing its name and the first item on the list which name starts from letters you typed will be selected.

Searching for items on local drives

When browsing a local folder, press Command + f to show the search panel. In it, you can specify the following options:

Option

Description
File Namea pattern to search in file names (with extensions)
Contenta pattern to search in content of files
Scopewhere the search should be performed, one of:
  • This Folder – search the current folder and its subfolders
  • This Mac – search all the local drives of this Mac
Exact Matchremoves “*” wildcards from the beginning and the end of the search patterns
Ignore Caseignore letter case when searching
No Spotlightif set, Spotlight will not be used, instead low-level search will be performed
Patterns for name and content search can use two wildcards:

Wildcard

Description
* means any character string, including the empty string
?means any character
For example, “*aa?cc*” would match “111aaBcc” and “1aa2cc3” but not “1bb2cc”.

When “Exact Match” is not set (default), the app automatically adds “*” at the beginning and the end of the pattern (for example, when you type “abc” the app will use “*abc*” as a search pattern).

Using Spotlight (default) is very fast because it bases on an index that Spotlight creates in advance. But Spotlight has its own policy what to index and what to skip which means that some file types and folders will never appear in its search results. It also skips items that have not been indexed yet. If you want to search all the files, you can switch to the low-level search. It takes much more time, but all the items are included in the operation. When searching in content of items without using Spotlight, “Exact Match” option is ignored.

Setting "File Name" or "Content" starts the search operation. Making a change in any attribute restarts it.

Searching for items on FTP/SFTP servers

When browsing FTP/SFTP folders, you can start search by pressing Command + f to show the search panel. In it, you can specify the following options:

Option

Description
File Namea pattern to search in file names (with extensions)
Exact Matchremoves “*” wildcards from the beginning and the end of a search pattern
Ignore Caseignore letter case when searching
You can use wildcards in file name pattern just like in searching on local drives.

Setting "File Name" starts the search operation. Making a change in any attribute restarts it.

Searching for items in archives

When browsing an archive file, you can start searching for its items by pressing Command + f. It shows the search panel where you can specify the following options:

Option

Description
File Namea pattern to search in file names (with extensions)
Exact Matchremoves “*” wildcards from the beginning and the end of a search pattern
Ignore Caseignore letter case when searching
You can use wildcards in file name pattern just like in searching on local drives.

Note that searching for items in an archive works only when you are browsing this archive – when searching local drives, items in archives are not taken into account.

Setting "File Name" starts the search operation. Making a change in any attribute restarts it.
