Using multiple FTP/SFTP connections

When running a download or upload operation in background or using multiple jobs, more than one connection to a remote FTP/SFTP server will be established. Be aware, that some servers, especially public anonymous FTP servers, limits number of concurrent connections that can be established by the given user or IP address. If this happens, some operations may fail with non-obvious errors. If you encounter such issue, you must stay with just one connection for the given FTP/SFTP server by not allowing operations to run in background or using multiple jobs.
