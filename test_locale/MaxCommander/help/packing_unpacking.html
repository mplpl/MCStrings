Packing and unpacking

Unpacking selected files

Unpacking selected files from an existing archive works in the same way as copying files between folders of a local drive.

Unpacking entire archives

You can also select one or more archive files and request the MaxCommander to unpack all of them, by pressing Option + Command + 6. By default, content of these archives will be put directly into the destination folder (which is what you set as the current folder in the target panel), but you can also mark ‘Create folder for each archive’ option to create a folder with name that matches the archive name, for each archive included in the operation, in the destination folder and unpack this archive there. Note that, if such folder already exists in the destination, the override rules like for a regular copy operation will apply.

The unpack operation can run in background and can use multiple jobs.

Adding files to existing archives

Packing files to an existing archive is similar to copying files between folders, but you have additional options to set in the File Copy Dialog. This allows you to set a compression level or encrypt newly added file with password (if the given type of archive supports it).

Packing to new archives

For creating a new archive, select the destination folder in one panel, then go to the other one, select the items to pack and press Option + Command + 5. This opens the Pack Dialog where you can set attributes of the new archive. In ‘To’ field set the name of the archive without extension and select the archive type from the list (which will also be used to automatically assign the file extension). You can also set the compression level and encrypt the archive with password by selecting options in ‘Options’ (you may need to unfold it). Setting the password is possible at two levels: for files or for entire archive. In the former case, you will be able to browse the archive without the password, but to display or unpack such files you will have to provide the password. In the latter case, the password will be required even for browsing the content of the archive. The encryption or changing compression level is only supported by some arhive types.

The pack operation can run in background. If another pack or delete operation is started for the same archive it is queued and will only proceed when on-going one is finished. It is not possible to set multiple jobs when packing but some compressors can take advantage of multi-core CPUs on their own - it happens fully automatic and does not require any configuration.

Alternative invocation

You can also request these operations by pressing a button on the Button Bar or the Touch Bar, selecting an item from the top menu, using drag and drop or using copy & paste.
