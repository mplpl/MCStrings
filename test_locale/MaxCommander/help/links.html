Creating links and aliases

Links allows you to have multiple items pointing to the same data (a file content) on a storage device. Creating links is not supported in archives or on remote file servers.

To create a link, select a single item in the current panel and have the destination for the new link selected in the other panel, then press Command + 9. It will open the Link Dialog where you can specify a name for your new link and select the link type. The following options are supported:

  • Symbolic Link (default) – standard POSIX symbolic link that can be created for files or folder.
  • Hard Link – low level link that can only be created for files of the same file system. After creating such like, it will not be distinguishable from the source but they both will use the same data chunk on a storage device (be careful with that!).
  • macOS Alias – macOS way of linking. It is similar to a symbolic link but is handled on the application level and therefore, some apps may not support it.

You can recognize symbolic links and aliases by looking at the icon – it has an arrow badge added – or by checking the ‘Size’ column – it will contain value. To distinguish symbolic links from aliases, check Item Info (press Command + i).

Hard links are not distinguishable for other files.

Alternative invocation

You can also request this operation by pressing a button on the Button Bar or the Touch Bar or selecting an item from the top menu.
